# دليل المساهمة

شكراً لرغبتك في المساهمة في The King AI! 🙏

## كيفية المساهمة

### تقارير الأخطاء

إذا وجدت خطأ، يرجى [فتح issue جديد](https://github.com/yourusername/king-ai/issues) مع:

- وصف واضح للمشكلة
- خطوات إعادة الإنتاج
- السلوك المتوقع
- السلوك الفعلي
- متصفحك وإصدار نظام التشغيل

### اقتراحات الميزات

لديك فكرة رائعة؟ [افتح issue جديد](https://github.com/yourusername/king-ai/issues) مع:

- وصف الميزة
- الفائدة المتوقعة
- أمثلة الاستخدام

### Pull Requests

1. **Fork المستودع**
2. **أنشئ فرع جديد**:
   ```bash
   git checkout -b feature/your-feature
   ```
3. **قم بالتغييرات**:
   - اتبع معايير الكود الموجودة
   - أضف تعليقات واضحة
   - تأكد من عدم كسر الميزات الموجودة

4. **اختبر التغييرات**:
   ```bash
   npm run dev
   npm run build
   ```

5. **Commit التغييرات**:
   ```bash
   git commit -m "Add: وصف واضح للتغيير"
   ```

6. **Push للفرع**:
   ```bash
   git push origin feature/your-feature
   ```

7. **افتح Pull Request** مع:
   - عنوان واضح
   - وصف التغييرات
   - رابط أي issues متعلقة

## معايير الكود

- استخدم ESLint (عند توفره)
- الأسماء الواضحة والمعبرة
- تعليقات عند الحاجة
- معايير React الحديثة

## نمط الـ Commits

```
type: وصف موجز

وصف مفصل (اختياري)

Fixes #issue_number
```

أنواع الـ Commits:
- `feat:` ميزة جديدة
- `fix:` إصلاح خطأ
- `docs:` تحديثات التوثيق
- `style:` تنسيق الكود
- `refactor:` إعادة هيكلة بدون تغيير السلوك
- `perf:` تحسينات الأداء
- `test:` إضافة أو تحديث الاختبارات

## الدعم

إذا كان لديك أسئلة:

- 💬 استخدم [GitHub Discussions](https://github.com/yourusername/king-ai/discussions)
- 📧 تواصل عبر البريد الإلكتروني
- 🐛 أبلغ عن الأخطاء في [Issues](https://github.com/yourusername/king-ai/issues)
